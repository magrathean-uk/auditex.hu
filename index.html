Auditex.
Tenant audit evidence,
kept local.

Auditex is an open-source Python-first CLI and MCP toolkit for Microsoft 365 and Google Workspace tenant audits. It gathers read-only evidence, normalizes it, and builds report packs operators can inspect before handoff.

View GitHub repo

Capabilities

Audit surfaces for real tenant work.

Auditex is built for operators who need a defensible read-only run, not a black-box score. Coverage is explicit, blocked checks are named, and raw tenant evidence remains on the operator machine.

Microsoft 365 audits

Run delegated or app-based read-only audits for identity, Conditional Access, audit logs, app consent, Exchange, SharePoint, devices, DNS, and secure-score evidence.

Google Workspace audits

Use domain-wide delegation or OAuth paths for Directory, Reports, Alert Center, Gmail settings, Drive metadata, Groups settings, Calendar sharing, devices, and domain posture.

No content-read posture

Audit collectors must not read Gmail bodies, Drive file content, Exchange mailbox bodies, SharePoint file content, or OneDrive file content.

Report packs

Generate normalized records, proof tables, API inventories, blockers, validation reports, customer packs, checksums, and handoff-ready summaries.

Probe-first runs

Check auth, profile fit, missing scopes, tenant policy blockers, licenses, and local tool readiness before deeper collection.

MCP and AI-safe context

Expose bundle facts through local MCP surfaces intended for evidence-cited reasoning, without secrets or unsupported claims.

Privacy

Raw evidence stays under operator control.

Auditex writes evidence to the selected local output directory. Raw evidence is not sent to Magrathean servers by default, and customer packs include checksums for handoff integrity.

Auditex is local-first and audit-only for the public product surface. Metadata and settings can still contain personal data such as names, email addresses, group membership, role assignments, device metadata, OAuth grant metadata, sharing metadata, and audit events. Treat every bundle as confidential.

Local evidenceRun bundles, reports, and handoff packs stay where the operator writes them.

Read-only proofAPI inventory records observed calls, permissions, read/write classification, and content-read status.

Auth materialService-account files, OAuth caches, access tokens, and secrets belong in local excluded paths.

Architecture

Python CLI. Stable bundle contract.

The product wrapper drives Microsoft 365 and Google Workspace collectors into the same local bundle contract, then report, export, compare, and MCP tools reason from saved evidence.

CLI and MCP

Operators run guided flows, direct CLI commands, compare, export, notify, report, and local MCP tools from their own environment.

Provider collectors

Microsoft Graph, Exchange adapters, Google Admin SDK, Reports, Gmail settings, Drive metadata, Calendar, Groups, and DNS collectors feed the same bundle shape.

Evidence DB

Normalized records and proof rows keep customer-facing claims tied to saved artifacts and exact limitations.

Validation

Read-only and no-content-read contracts fail unsafe bundles before handoff.
