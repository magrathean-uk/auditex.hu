Quickstart

First run, kept simple.

This page exists for the operator who wants to know what to do next, without losing the read-only and local-first contract on the first click.

Download a release

Use the GitHub release archive and checksum file from the download page.

Verify the release

Check the archive hash before unpacking or placing the tool in an internal distribution path.

Pick auth mode

Choose delegated, app-based, domain-wide delegation, or OAuth path based on tenant policy and least-surprise operator workflow.

Run probe-first checks

Check scopes, tenant blockers, licensing gaps, and environment readiness before deeper collection.

First run checklist

Keep the first run boring and reviewable.

  1. Download release archive and `SHA256SUMS`.
  2. Verify checksum locally.
  3. Choose output directory on an operator-controlled machine.
  4. Configure provider auth path.
  5. Run probe-first checks.
  6. Run first audit collection.
  7. Inspect API inventory, blockers, validation, and report pack.

What to inspect after the first run

Do not skip the proof artifacts.

API inventory

Check observed API usage, permissions, and read/write classification.

Validation

Confirm the bundle still satisfies the public read-only and no-content-read position.

Blockers

Coverage gaps and provider restrictions should be visible, not hidden in a score.

Report pack

Use report packs as review material, not as a substitute for human judgement.

First run FAQ

Answer the trust questions on the docs page too.

Does Auditex read content?

No for the public audit surface. It is designed to stay read-only and no-content-read for Gmail bodies, Drive file content, Exchange mailbox body content, SharePoint file content, and OneDrive file content.

Is a first run a certification result?

No. Auditex is not certification, not legal advice, and not guaranteed security. It collects evidence for human review.

What if the run is blocked?

Blocked checks should stay in the bundle and in the report narrative so the operator can explain the limit honestly.
