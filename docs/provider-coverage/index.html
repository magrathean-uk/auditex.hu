Provider coverage

Coverage page with explicit limits.

Coverage pages matter because operators need to know what will be attempted, what stays out of scope, and what the public read-only surface will not do.

Provider Surface Audit use Boundary
Microsoft 365 Identity, Conditional Access, app consent Tenant posture and access control evidence Read-only posture only
Microsoft 365 Exchange settings, SharePoint, OneDrive, devices Settings, metadata, sharing posture, device posture No mailbox body or file-content reads
Microsoft 365 Audit logs, DNS, secure score Timeline evidence and control posture Findings still need human review
Google Workspace Directory, Reports, Alert Center Directory posture, reporting, and admin alerts Read-only posture only
Google Workspace Gmail settings, Drive metadata, Groups, Calendar Settings and sharing metadata collection No Gmail body or Drive file-content reads
Google Workspace Devices and domain posture Device and tenant-wide administrative context Blocked checks should remain visible

Coverage means scope and limits both.
